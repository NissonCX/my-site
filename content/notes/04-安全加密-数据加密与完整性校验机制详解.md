## 一、政务系统为什么要加密？

### 1.1 现实场景

```
检察院系统里存储的数据：
├─ 公务员的手机号
├─ 举报人的姓名、联系方式
├─ 案件相关的敏感信息
└─ 身份证号、银行账户

风险：
  1. 数据库被攻击 → 信息泄露
  2. 内部人员偷偷改数据库 → 无法发现
  3. 不符合《密码法》要求
```

### 1.2 加密的三个层面

| 层面 | 作用 | 方式 |
|------|------|------|
| **传输加密** | 防止数据在网上传输时被窃听 | HTTPS (TLS 1.2+) |
| **存储加密** | 防止数据库被盗后信息泄露 | SM4 / 三未加密 |
| **完整性校验** | 防止数据被偷偷篡改 | HMAC（哈希消息认证码） |

## 二、SM4 vs 三未加密

### 2.1 SM4 - 国密对称加密算法

```
SM4 = 国家密码局制定的对称加密算法
    = 类似于国际标准的 AES

特点：
  ├─ 对称加密：加密和解密用同一个密钥
  ├─ 国密标准：中国政府要求政务系统必须用国密
  └─ 软件实现：纯代码，不需要额外硬件

类比：
  SM4 是中国的 AES
  就像 GB18030 是中国的 UTF-8
```

### 2.2 三未 - 硬件加密机

```
三未 = 一家做国密硬件的公司
    = 生产"加密机"这种硬件设备

特点：
  ├─ 硬件加密：密钥存在硬件里，拿不出来
  ├─ 安全性更高：即使代码泄露，密钥也安全
  ├─ 符合密评要求：国家密码管理局的认证
  └─ 需要花钱：购买硬件设备

类比：
  三未加密机 = 银行的 U 盾
  密钥在硬件里 = 私钥在 U 盾里，偷不走
```

### 2.3 如何选择？

| 方案 | 密钥来源 | 优点 | 缺点 | 适用场景 |
|------|---------|------|------|---------|
| **SM4** | 硬编码在代码中 | 免费、快速 | 密钥可能泄露 | 一般敏感数据 |
| **三未** | 硬件加密机 | 密钥无法泄露、符合密评要求 | 需要花钱、调用慢 | 核心敏感数据 |

**本项目选择**：核心数据用三未（SysUser 的敏感字段）

## 三、TypeHandler 自动加解密机制

### 3.1 问题：如何让加密"自动化"？

**不用 TypeHandler 的痛苦**：
```java
// 每次保存都要手动加密
user.setPhone(SM4Utils.encrypt("13800138000"));
user.setName(SM4Utils.encrypt("张三"));
userMapper.insert(user);

// 每次查询都要手动解密
User user = userMapper.selectById(1);
user.setPhone(SM4Utils.decrypt(user.getPhone()));
```

**问题**：
- ❌ 容易遗漏：某次忘记加密就存明文了
- ❌ 代码重复：每个字段都要写加密/解密
- ❌ 维护困难：字段多了很乱

### 3.2 用 TypeHandler 的好处

```java
// 实体类配置一次
@TableName("sys_user")
public class SysUser {
    @TableField(typeHandler = SansecTypeHandler.class)
    private String phone;  // 这个字段自动加密/解密
}

// 业务代码完全不用管加密
user.setPhone("13800138000");  // 直接传明文
userMapper.insert(user);        // 自动加密存数据库

User user = userMapper.selectById(1);
System.out.println(user.getPhone());  // 自动解密，输出明文
```

### 3.3 TypeHandler 的本质

```
TypeHandler = MyBatis 的"中间人"

作用：
  ├─ 写入数据库前：自动调用加密方法
  ├─ 读取数据库后：自动调用解密方法
  └─ 对业务代码透明（业务代码不需要知道）
```

### 3.4 完整流程

## 四、Spring 注入技巧：SansecCryptoServiceHolder

### 4.1 为什么需要这个类？

**问题**：TypeHandler 无法直接使用 @Autowired

```java
public class SansecTypeHandler extends BaseTypeHandler<String> {

    // ❌ 这样能工作吗？
    @Autowired
    private ISansecCryptoService cryptoService;

    // 答案：不能！
}
```

**原因**：

```
MyBatis 的 TypeHandler 不是 Spring 管理的 Bean！

原因：
  ├─ TypeHandler 是 MyBatis 框架通过反射创建的
  ├─ 不是通过 @Component 扫描创建的
  ├─ 没有 Spring 的依赖注入功能
  └─ @Autowired 不生效 → cryptoService = null
```

### 4.2 解决方案：静态持有者模式

```java
@Component  ← Spring 管理这个类
public class SansecCryptoServiceHolder {

    static ISansecCryptoService instance;  ← static 变量，全局唯一

    @Autowired  ← Spring 自动调用这个构造函数
    public SansecCryptoServiceHolder(ISansecCryptoService cryptoService) {
        SansecCryptoServiceHolder.instance = cryptoService;  ← 注入进来
    }
}
```

### 4.3 @Autowired 在构造函数上的含义

**简单说**：Spring 创建这个类时，自动调用这个构造函数，并自动找参数填进去。

```
Spring 启动时：
  1. 扫描到 @Component
  2. 找到构造函数：SansecCryptoServiceHolder(ISansecCryptoService)
  3. 找到 ISansecCryptoService 的实现类：SansecServiceV6Impl
  4. 创建 SansecServiceV6Impl 实例
  5. 调用构造函数：new SansecCryptoServiceHolder(sansecServiceV6Impl)
  6. instance = sansecServiceV6Impl
```

### 4.4 完整的依赖注入链

```
┌─────────────────────────────────────────────────────────────┐
│                    依赖注入链路                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【Spring 管理的 Bean】                                       │
│                                                             │
│  ISansecCryptoService (接口)                                 │
│     │                                                       │
│     ▼                                                       │
│  SansecServiceV6Impl (实现类)                                │
│     ├─ 包含配置（加密机地址、证书等）                          │
│     ├─ 由 Spring 创建和管理                                   │
│     └─ @Service                                             │
│                                                             │
│  【桥接类】                                                   │
│                                                             │
│  SansecCryptoServiceHolder (@Component)                      │
│     ├─ static instance                                      │
│     ├─ 构造函数注入 ISansecCryptoService                     │
│     └─ 把 Spring Bean 赋值给 static 变量                     │
│                                                             │
│  【MyBatis 管理的类】                                         │
│                                                             │
│  SansecTypeHandler (不是 @Component)                         │
│     ├─ MyBatis 创建，不是 Spring 创建                          │
│     ├─ 无法使用 @Autowired                                   │
│     ├─ 通过 SansecCryptoServiceHolder.instance 访问          │
│     └─ 拿到 Spring 管理的加密服务                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.5 核心设计思想（重要理解）

> **"利用 @Component 和 @Autowired 巧妙完成了 SansecCryptoServiceHolder 数据注入，在 handler 无法利用 Spring Bean 容器的情景完成了加解密服务实现的注入。"**

## 五、HMAC 完整性校验

### 5.1 什么是 HMAC？

```
HMAC = Hash-based Message Authentication Code
     = 基于哈希的消息认证码

作用：
  ├─ 验证数据是否被篡改
  ├─ 保证数据完整性
  └─ 防止内部人员偷偷改数据库
```

### 5.2 HMAC 的工作原理

```
原始数据：张三, 13800138000, 500110199001011234
    │
    ▼
HMAC 计算：
  ├─ 使用密钥（硬件加密机）
  ├─ SM3 杂凑算法（国密标准）
  └─ 生成：hmac = "abc123..."
    │
    ▼
存储：
  ├─ 数据：张三, 13800138000, 500110199001011234 (加密)
  └─ HMAC：abc123... (加密)

验证时：
  ├─ 重新计算 HMAC
  └─ 对比存储的 HMAC（不匹配则检测到篡改）
```

### 5.3 项目中的使用

```java
@TableName("sys_user")
public class SysUser {

    @HmacField(order = 1)  // 用户名参与 HMAC 计算
    private String username;

    @HmacField(order = 2)  // 密码参与 HMAC 计算
    private String password;

    @HmacField(order = 4)  // 手机号参与 HMAC 计算
    private String phone;

    // order = 1, 2, 4 → 拼接顺序：username + password + phone
}
```

### 5.4 登录时的完整性校验

**代码位置**：`AbstractAuthTemplate.java:87-92`

```java
// 完整性校验
String plainText = HmacPlainBuilder.build(user);  // 拼接字段
boolean verify = cryptoService.hmacVerify(plainText, user.getHmacResult());
if (!verify) {
    loginLogService.loginFaiLog(user);
    throw new BizException(BizRespCode.ERR_2007);  // 完整性校验失败
}
```
