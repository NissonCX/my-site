## 一、工厂模式详解

### 1.1 核心原理

```java
@Service
public class AuthServiceFactory {

    // Spring 自动注入所有实现类
    @Resource
    private List<IAuthService> authServiceList;

    // 根据 authType 路由到具体实现
    public LoginTokenVO appLogin(LoginRequest request) {
        return authServiceList.stream()
            .filter(x -> x.getAuthType().equals(request.getAuthType()))
            .findFirst()
            .orElseThrow()
            .appLogin(request);
    }
}
```

### 1.2 工作流程

```
┌─────────────────────────────────────────────────────────┐
│                     Spring 启动                          │
├─────────────────────────────────────────────────────────┤
│  扫描 @Service                                        │
│  发现所有 IAuthService 实现类                            │
│  ┌───────────────────────────────────────────┐           │
│  │ YkbAuthServiceImpl   implements IAuthService │           │
│  │ YkzAuthServiceImpl   implements IAuthService │           │
│  │ SysUserAuthServiceImpl implements IAuthService │         │
│  │ HAuthServiceImpl   implements IAuthService │           │
│  └───────────────────────────────────────────┘           │
│           │                                               │
│           ▼                                               │
│  自动组装成 List<IAuthService>                             │
│           │                                               │
│           ▼                                               │
│  注入给 AuthServiceFactory                                │
└─────────────────────────────────────────────────────────┘
```

### 1.3 运行时路由

```
请求: {"authType": "YKB"}
    │
    ▼
┌────────────────────────────────────────┐
│  authServiceList.stream()              │
│     .filter(x -> x.getAuthType()      │
│             .equals("YKB"))         │ ← 过滤
│     .findFirst()                       │
└────────────────────────────────────────┘
    │
    ▼
YkbAuthServiceImpl
    │
    ▼
.appLogin(request)
```

### 1.4 关键点

| 实现类 | getAuthType() 返回值 |
|--------|---------------------|
| YkbAuthServiceImpl | AuthTypeEnum.YKB |
| YkzAuthServiceImpl | AuthTypeEnum.YKZ |
| SysUserAuthServiceImpl | AuthTypeEnum.DEFAULT |

## 二、抽象模板模式详解

### 2.1 结构设计

```
         AbstractAuthTemplate (抽象父类)
                    │
        ┌───────────┴───────────┐
        │                   │
   公用方法             抽象方法
        │                   │
    generateToken()    getOrCreateUser()
        │                   │
        │                   └─── 子类实现
        │
        ▼
    JWT + Session 生成
```

### 2.2 代码实现

```java
// 父类：公用逻辑
public abstract class AbstractAuthTemplate implements IAuthService {

    // 公用方法：生成 token + session
    public LoginTokenVO generateToken(SysUser user, List<SysRole> roles, AuthTypeEnum authType) {
        StpUtil.login(user.getId());              // 生成 JWT
        UserContext.setUserToken(userToken);     // 存入 Session
        SaTokenInfo tokenInfo = StpUtil.getTokenInfo();
        dto.setAccessToken(tokenInfo.getTokenValue());
        return dto;
    }

    // 抽象方法：子类必须实现
    public abstract SysUser getOrCreateAppUser(LoginRequest request);
    public abstract SysUser getOrCreateManageUser(ManageLoginDTO request);
}

// 子类：实现差异化登录逻辑
@Component
public class YkbAuthServiceImpl extends AbstractAuthTemplate {
    @Override
    public AuthTypeEnum getAuthType() {
        return AuthTypeEnum.YKB;
    }

    @Override
    public SysUser getOrCreateAppUser(LoginRequest request) {
        // 渝快办特有的登录逻辑
        // 1. 调用渝快办接口验证
        // 2. 查询或创建本地用户
        return user;
    }
}
```

### 2.3 调用链路

```
客户端请求
    │
    ▼
AuthController.login(/sys/auth/app/login)
    │
    ▼
AuthServiceFactory.appLogin(request)
    │
    ├─ 根据 authType="YKB"
    │
    ▼
YkbAuthServiceImpl.appLogin(request)
    │
    ├─ getOrCreateAppUser()        [子类实现]
    │
    └─ generateToken()             [父类公用]
        │
        ├─ StpUtil.login()        → 生成 JWT
        └─ UserContext.setUserToken() → 存 Session
```

## 五、总结

### 工厂模式
- **启动时**：Spring 自动收集所有实现类到 List
- **运行时**：根据 authType 过滤出需要的实现类

### 抽象模板模式
- **父类**：放公用逻辑（生成 token + session）
- **子类**：放差异化逻辑（不同登录方式）

### 配合使用
```
工厂路由 → 找到子类 → 子类调用父类公用逻辑
```
