## 一、渝快办 vs 渝快政对比

| 对比维度 | 渝快办（YKB） | 渝快政（YKZ） |
|---------|-------------|-------------|
| **用户群体** | C端市民 | G端公务员 |
| **API 调用次数** | 1次 | 3-5次 |
| **SDK** | 自研 GatewaySender | 阿里 ExecutableClient |
| **权限验证** | ❌ 无 | ✅ 组织白名单（向上递归） |
| **调试模式** | ❌ 无 | ✅ debug / baffle 双开关 |
| **加密方式** | AES | SM2（国密） |
| **用户白名单** | ❌ 无 | ✅ whiteUid |
| **组织白名单** | ❌ 无 | ✅ whiteOrgan（向上遍历） |
| **角色** | ORDINARY（普通用户） | PROCURATORATE（检察院） |

## 三、双 Token 架构

### 3.1 双系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      YKZ 双系统架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   SSO 系统（阿里网关）                用户中心（政务系统）      │
│      │                                    │                 │
│      ├─ ssoAccessToken                   ├─ userCenterToken  │
│      │  获取：appKey+appSecret(明文)     │  获取：appId+SM2  │
│      │  用途：换取用户信息                 │  用途：查询详情     │
│      │  标准：OAuth2互联网标准             │  标准：国密标准     │
│      │                                    │                 │
│      └─ 返回：accountId                  └─ 返回：完整用户数据 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 认证链

| 步骤 | 需要的信息 | 证明的内容 |
|------|-----------|-----------|
| 1 | appKey + appSecret | 证明"我是合法应用" |
| 2 | ssoAccessToken + code | 证明"用户已授权" |
| 3 | appId + SM2(appSecret) | 证明"我能访问用户中心" |
| 4 | accountId + userCenterToken | 证明"我有权查询该用户" |

## 四、debug vs baffle 开关

### 4.1 代码实现

```java
// debug 模式：code长度≤20 当作用户名
if (ykzConfig.getDebug() && StringUtils.length(authCode) <= 20) {
    return sysUserService.getUserByUserName(authCode);
}

// baffle 模式：所有 code 都当用户名（挡板模式）
if (ykzConfig.getBaffle()) {
    return sysUserService.getUserByUserName(authCode);
}
```

### 4.2 开关对比

| 开关 | 触发条件 | 作用 |
|------|---------|------|
| `debug` | `debug=true && code长度≤20` | 开发调试：直接用用户名登录 |
| `baffle` | `baffle=true` | 挡板模式：当 YKZ 服务不可用时使用 |

### 4.3 风险与改进

**风险**：
- 配置误带到生产环境，任何人可登录

**改进**：
- 用 `@Profile` 限制环境
- 加审计日志
- 加 IP 白名单
- 绝不在业务代码里写 debug 逻辑，用 AOP

@Override
public SysUser getOrCreateAppUser(LoginRequest model) {

### 5.3 获取 SSO AccessToken

```java
private String requestAccessToken() {
    // 只用 appKey + appSecret，没有 code！
    IntelligentGetClient client = executableClient.newIntelligentGetClient("/gettoken.json");
    OapiGettokenJsonRequest request = new OapiGettokenJsonRequest();
    request.setAppkey(ykzConfig.getAppKey());
    request.setAppsecret(ykzConfig.getAppSecret());

    OapiGettokenJsonResponse response = client.get(request);
    return JSONObject.parseObject(response.getContent().getData()).getString("accessToken");
}
```

**关键点**：这是应用级 Token，所有用户共享，不是用 code 换的！

### 5.4 用 code 换 accountId

```java
public YkzSsoModel ssoAuth(String authCode) {
    IntelligentPostClient client = executableClient.newIntelligentPostClient("/rpc/oauth2/dingtalk_app_user.json");
    OapiRpcOauth2DingtalkAppUserJsonRequest request = new OapiRpcOauth2DingtalkAppUserJsonRequest();
    request.setAccess_token(requestAccessToken());  // SSO Token
    request.setAuth_code(authCode);                 // 用户授权码

    OapiRpcOauth2DingtalkAppUserJsonResponse response = client.post(request);
    return content.getJSONObject("content").getObject("data", YkzSsoModel.class);
}

// YkzSsoModel 返回数据
{
    "accountId": "123456",    // 核心字段
    "account": "xxx",
    "lastName": "张三",
    "employeeCode": "E001"
}
```

### 5.5 获取用户中心 Token - 国密加密

```java
public String userCenterAccessToken() {
    JSONObject param = new JSONObject()
        .fluentPut("data", new JSONObject()
            .fluentPut("appId", ykzConfig.getUserCenter().getAppId())
            .fluentPut("appSecret", SM2ToolUtil.sm2Encode(jwtHelper.getPublicKey(),
                                                          ykzConfig.getUserCenter().getAppSecret()))
        ).fluentPut("requestId", IdUtil.randomUUID());

    YkzBaseRespModel resp = doJsonPost(ykzConfig.getUserCenter().getPath() + "/ykz/access_token", param);
    return JSONObject.parseObject(resp.getData()).getString("accessToken");
}
```

**关键点**：appSecret 使用 SM2 国密加密！

## 七、常见面试题

### Q1: 为什么 SSO 和用户中心的认证方式不同？

**A**:
- **SSO 系统**（阿里网关）：遵循 OAuth2 互联网标准，HTTPS 传输足够
- **用户中心**（政务系统）：必须符合国家密码法规，使用 SM2 国密加密
- 本质是两套系统：阿里网关（互联网标准）+ 政务用户中心（国密标准）

### Q2: debug 和 baffle 开关有什么风险？

**A**:
- **风险**：配置误带到生产环境，任何人可登录
- **改进**：
  1. 用 `@Profile` 限制环境
  2. 加审计日志
  3. 加 IP 白名单
  4. 绝不在业务代码里写 debug 逻辑，用 AOP

### Q3: 向上遍历组织白名单有什么风险？

**A**:
- **性能问题**：每层一次 HTTP 调用，组织深则慢
- **级联失败**：组织 API 挂了，整个登录失败
- **优化方案**：
  1. 一次性返回完整组织链
  2. 本地缓存组织树（Redis）
  3. 让上游返回 `isInWhiteList` 字段

## 八、总结

### 核心认知

| 步骤 | 关键词 |
|------|--------|
| 1 | **appKey+appSecret** → 证明应用合法性 |
| 2 | **ssoToken+code** → 获取 accountId |
| 3 | **appId+SM2** → 获取用户中心 Token |
| 4 | **accountId+Token** → 获取用户数据 |
| 5 | **组织白名单向上遍历** → 验证用户权限 |

### 一句话总结

> **SSO 系统负责身份认证，用户中心负责信息管理**
> **两个系统独立认证，各自遵循不同的安全标准**
