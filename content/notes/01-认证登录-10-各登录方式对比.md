## 一、三平台完整对比

| **加密方式** | AES | AES | SM2（国密） |

## 二、通用登录流程抽象

### 2.1 所有三方登录的通用流程

```
┌─────────────────────────────────────────────────────────────────────┐
│              所有三方登录的通用流程                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  第1步：前端获取 code（所有平台都一样）                               │
│     ├─ 微信：wx.login() → code                                     │
│     ├─ 渝快办：跳转授权页 → code                                     │
│     └─ 渝快政：跳转授权页 → code                                     │
│                                                                     │
│  第2步：前端发送 code 给后端（所有平台都一样）                        │
│     POST /app/login { code, authType }                              │
│                                                                     │
│  第3步：后端用 code 换取用户标识（平台差异）                          │
│     ├─ 微信：code + appid + secret → openid + session_key          │
│     ├─ 渝快办：code + appKey + secret → 用户信息（含 id）            │
│     └─ 渝快政：code + ssoToken → accountId → 用户中心               │
│                                                                     │
│  第4步：后端创建/绑定用户（所有平台都一样）                           │
│     用用户标识查询本地数据库                                         │
│     ├─ 存在 → 返回用户                                              │
│     └─ 不存在 → 创建用户                                            │
│                                                                     │
│  第5步：生成 JWT 返回（所有平台都一样）                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 工厂模式统一处理

```java
// 登录请求统一入口
@PostMapping("/app/login")
public R<LoginTokenVO> appLogin(@RequestBody LoginRequest request) {
    // request.authType 区分：WE_CHAT / YKB / YKZ
    LoginTokenVO token = authServiceFactory.appLogin(request);
    return R.ok(token);
}

// 工厂根据 authType 路由
public LoginTokenVO appLogin(LoginRequest request) {
    return authServiceList.stream()
        .filter(s -> s.getAuthType().equals(request.getAuthType()))
        .findFirst()
        .orElseThrow()
        .appLogin(request);
}
```

## 三、平台特色功能

### 3.1 微信特色：加密手机号

```
微信独有的加密机制：

前端：
  ├─ wx.login() → code
  ├─ <button open-type="getPhoneNumber"> → encryptedData + iv

后端：
  ├─ code → openid + session_key
  ├─ session_key + iv + encryptedData → 解密手机号
  └─ 手机号创建/查询用户

安全设计：
  ├─ session_key 只在后端
  ├─ 前端无法解密手机号
  └─ 防止恶意小程序窃取用户手机号
```

### 3.2 渝快办特色：JSAPI 预初始化

```
渝快办独有的预初始化机制：

前端启动时：
  ├─ GET /ykb/jsapiToken → {jsapiToken, appId, scopes}
  ├─ YKB.init({...}) → JS-SDK 初始化

用户点击登录：
  ├─ YKB.login() → 跳转授权页 → code
  └─ POST /app/login {code}

设计原因：
  ├─ JS-SDK 是外挂的，需要手动初始化
  ├─ jsapiToken 有时效性（2小时）
  └─ 应用启动时获取，用户点击立即跳转
```

### 3.3 渝快政特色：组织白名单

```
渝快政独有的权限验证机制：

登录流程：
  ├─ SSO 认证 → accountId
  ├─ 用户中心查询 → 用户信息 + 岗位列表
  ├─ 组织白名单向上遍历 → 权限验证
  └─ 通过 → 创建用户 → 生成 JWT

白名单设计：
  ├─ whiteUid：特殊用户直接通过
  ├─ whiteOrgan：普通用户检查组织
  └─ 向上遍历：降低维护成本

设计原因：
  ├─ 政务系统只给特定组织的人使用
  ├─ 用户可能属于任何下级部门
  └─ 新增部门不需要改配置
```

## 四、选择建议

### 4.1 场景匹配

| 场景 | 推荐平台 | 原因 |
|------|---------|------|
| **C端市民应用** | 渝快办（YKB） | 重庆市民实名认证 |
| **G端政务应用** | 渝快政（YKZ） | 公务员组织管理 |
| **通用小程序** | 微信 | 覆盖面广，流程简单 |

### 4.2 技术选型考虑

| 考虑点 | 微信 | 渝快办 | 渝快政 |
|--------|------|--------|--------|
| **开发复杂度** | 低 | 中 | 高 |
| **API 调用** | 1次 | 1次 | 3-5次 |
| **需要初始化** | ❌ | ✅ | ❌ |
| **权限控制** | 自己实现 | 自己实现 | 平台提供 |
| **安全标准** | AES | AES | 国密 SM2 |

## 五、总结

### 5.1 核心认知

| 平台 | 特点 | 适用场景 |
|------|------|---------|
| **微信** | 简单、标准化 | 通用小程序 |
| **渝快办** | 市民实名认证 | C端政务应用 |
| **渝快政** | 公务员、组织管理、国密 | G端政务应用 |

### 5.2 一句话总结

> **微信最简单（1次调用），渝快办需初始化（JSAPI），渝快政最复杂（双系统+国密+白名单）**
