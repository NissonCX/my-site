## 一、概述

项目使用自定义 `@Log` 注解记录用户的操作日志，通过 AOP 切面自动捕获请求信息并异步保存到数据库。

---

## 二、注解定义

位置：`cn.cqchain.mp.app.annotation.Log`

```java
@Target({ElementType.PARAMETER, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface Log {
    /** 模块名称 */
    String title() default "";

    /** 业务操作类型 */
    BusinessType businessType() default BusinessType.OTHER;

    /** 操作人类别 */
    OperatorType operatorType() default OperatorType.MANAGE;

    /** 是否保存请求参数 */
    boolean isSaveRequestData() default true;

    /** 是否保存响应参数 */
    boolean isSaveResponseData() default true;

    /** 排除指定的请求参数 */
    String[] excludeParamNames() default {};
}
```

## 三、业务类型枚举

位置：`cn.cqchain.mp.app.enums.BusinessType`

| 枚举值 | 说明 | 使用场景 |
|--------|------|---------|
| `OTHER` | 其他 | 默认值 |
| `INSERT` | 新增 | 新增数据操作 |
| `UPDATE` | 修改 | 修改数据操作 |
| `DELETE` | 删除 | 删除数据操作 |
| `GRANT` | 授权 | 权限分配操作 |
| `EXPORT` | 导出 | 导出数据操作 |
| `IMPORT` | 导入 | 导入数据操作 |
| `FORCE` | 强退 | 强制用户下线 |
| `GENCODE` | 生成代码 | 代码生成操作 |
| `CLEAN` | 清空数据 | 清空缓存或数据 |

## 四、使用方式

### 4.1 导入注解

```java
import cn.cqchain.mp.app.annotation.Log;
import cn.cqchain.mp.app.enums.BusinessType;
```

### 4.2 在Controller方法上添加注解

```java
/**
 * 新增敏感词
 */
@Log(title = "敏感词管理", businessType = BusinessType.INSERT)
@PostMapping(value = "/add")
@Operation(summary = "新增敏感词")
public R<Void> add(@Validated @RequestBody SysSensitiveWordBo bo) {
    // ...
}

/**
 * 编辑敏感词
 */
@Log(title = "敏感词管理", businessType = BusinessType.UPDATE)
@PostMapping(value = "/edit")
@Operation(summary = "编辑敏感词")
public R<Void> edit(@Validated @RequestBody SysSensitiveWordBo bo) {
    // ...
}

/**
 * 删除敏感词
 */
@Log(title = "敏感词管理", businessType = BusinessType.DELETE)
@PostMapping("/remove/{ids}")
@Operation(summary = "敏感词删除")
public R<Void> remove(@PathVariable String[] ids) {
    // ...
}
```

### 4.3 导入导出操作

```java
/**
 * 导出模版
 */
@Log(title = "模版管理", businessType = BusinessType.EXPORT)
@PostMapping("/export")
@Operation(summary = "导出模版")
public void export(SysTemplateBo bo, HttpServletResponse response) {
    // ...
}

/**
 * 导入模版
 */
@Log(title = "模版管理", businessType = BusinessType.IMPORT)
@PostMapping(value = "/import", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
@Operation(summary = "导入模版")
public R<Void> importExcel(@RequestPart("file") MultipartFile file) {
    // ...
}
```

## 五、注解位置

注解应放在方法注释之后、`@PostMapping`/`@GetMapping` 之前：

```java
/**
 * 方法说明
 */
@Log(title = "模块名称", businessType = BusinessType.XXX)
@PostMapping("/path")
@Operation(summary = "接口说明")
public R<Void> method() {
    // ...
}
```

## 六、注意事项

1. **title 命名规范**：使用模块名称，如"敏感词管理"、"模版管理"、"角色认证申请"
2. **businessType 选择**：根据操作类型选择对应的枚举值
3. **无需添加的场景**：查询类接口（GET请求）不需要添加日志注解
4. **需要添加的场景**：新增(INSERT)、修改(UPDATE)、删除(DELETE)、导入(IMPORT)、导出(EXPORT)

---

## 七、实现原理

日志注解通过 AOP 切面实现：

1. `OpLogAspect` 切面类拦截所有带 `@Log` 注解的方法
2. 通过事件发布 `OperLogEvent` 异步处理日志记录
3. `SysOperLogService` 负责日志数据的持久化

核心流程：
```
Controller方法执行 → AOP拦截 → 发布日志事件 → 异步保存日志
```
