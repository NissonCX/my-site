## 一、模块定位

**通过菜单权限控制访问**，谁有会话管理的菜单，谁就能用这些接口。

---

## 二、接口清单汇总

### 2.1 用户登录会话信息

| 功能 | 路径 | 方法 | 说明 |
|------|------|------|------|
| 查看我的设备列表 | `/app/session/my-devices` | GET | 用户查看自己的登录设备 |
| 查看我的当前会话 | `/app/session/current` | GET | 用户查看当前会话详情 |
| 查看指定用户会话 | `/admin/session/user/{userId}` | GET | 有权限的人查看指定用户 |

### 2.2 设置会话有效期

| 功能 | 路径 | 方法 | 说明 |
|------|------|------|------|
| 获取会话有效期配置 | `/admin/session/timeout` | GET | 获取当前token有效期设置 |
| 设置会话有效期 | `/admin/session/timeout` | PUT | 设置token有效期 |

### 2.3 设置会话活跃状态

| 功能 | 路径 | 方法 | 说明 |
|------|------|------|------|
| 踢人下线 | `/admin/session/kickout/{userId}` | DELETE | 让用户下线 |
| 顶号 | `/admin/session/replace/{userId}` | DELETE | 只保留最新登录 |
| 退出当前登录 | `/app/logout` | POST | 当前用户退出 |

### 2.4 可视化界面展示

| 功能 | 路径 | 方法 | 说明 |
|------|------|------|------|
| 在线用户列表 | `/admin/session/online-list` | GET | 分页获取在线用户列表 |

### 2.5 查看会话情况

| 功能 | 路径 | 方法 | 说明 |
|------|------|------|------|
| 查看历史登录记录 | `/app/session/history` | GET | 用户查看自己的登录历史 |

### 2.6 统计在线数量

| 功能 | 路径 | 方法 | 说明 |
|------|------|------|------|
| 获取在线统计 | `/admin/session/statistics` | GET | 获取在线人数等统计数据 |

## 三、接口详情

### 3.1 查看我的设备列表

**接口**: `GET /app/session/my-devices`

**说明**: 用户查看自己当前登录的设备列表

**响应示例**:
```json
{
  "code": 200,
  "data": [
    {
      "deviceId": "token值",
      "deviceType": "PC",
      "browser": "Chrome",
      "ipAddress": "192.168.1.100",
      "loginTime": "2026-03-09T10:00:00",
      "lastActiveTime": "2026-03-09T14:00:00"
    },
    {
      "deviceId": "token值2",
      "deviceType": "Mobile",
      "browser": "Safari",
      "ipAddress": "192.168.1.101",
      "loginTime": "2026-03-09T09:00:00",
      "lastActiveTime": "2026-03-09T13:00:00"
    }
  ]
}
```

---

### 3.2 查看我的当前会话

**接口**: `GET /app/session/current`

**说明**: 用户查看当前登录的会话详情

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "deviceId": "token值",
    "deviceType": "PC",
    "browser": "Chrome",
    "os": "Windows 11",
    "ipAddress": "192.168.1.100",
    "loginTime": "2026-03-09T10:00:00",
    "lastActiveTime": "2026-03-09T14:00:00"
  }
}
```

---

### 3.3 查看指定用户会话

**接口**: `GET /admin/session/user/{userId}`

**说明**: 有权限的人查看指定用户的会话（在线+历史）

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "userId": "123456",
    "username": "张三",
    "onlineCount": 2,
    "devices": [
      {
        "deviceId": "token值",
        "deviceType": "PC",
        "loginTime": "2026-03-09T10:00:00",
        "isCurrent": true
      }
    ]
  }
}
```

---

### 3.4 获取/设置会话有效期

**获取**: `GET /admin/session/timeout`

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "defaultTimeout": 86400,
    "roleConfigs": [
      { "roleCode": "student", "timeout": 604800 },
      { "roleCode": "teacher", "timeout": 259200 }
    ]
  }
}
```

**设置**: `PUT /admin/session/timeout`

**请求体**:
```json
{
  "defaultTimeout": 86400,
  "roleConfigs": [
    { "roleCode": "student", "timeout": 604800 }
  ]
}
```

---

### 3.5 踢人下线

**接口**: `DELETE /admin/session/kickout/{userId}`

**说明**: 让指定用户下线（踢出所有会话）

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |

**响应示例**:
```json
{
  "code": 200,
  "msg": "操作成功"
}
```

---

### 3.6 顶号

**接口**: `DELETE /admin/session/replace/{userId}`

**说明**: 只保留最新登录，其他设备全部踢出

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userId | String | 是 | 用户ID |

**响应示例**:
```json
{
  "code": 200,
  "msg": "操作成功"
}
```

---

### 3.7 退出登录

**接口**: `POST /app/logout`

**说明**: 当前用户退出登录

**响应示例**:
```json

  "code": 200,
  "msg": "退出成功"
}
```

### 3.8 在线用户列表

**接口**: `GET /admin/session/online-list`

**说明**: 分页获取当前在线用户列表

**参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认1 |
| size | int | 否 | 每页条数，默认10 |
| keyword | String | 否 | 搜索用户名 |

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "total": 100,
    "list": [
      {
        "userId": "123456",
        "username": "张三",
        "deviceType": "PC",
        "ipAddress": "192.168.1.100",
        "loginTime": "2026-03-09T10:00:00"
      }
    ]
  }
}
```

## 五、技术说明

**项目使用Sa-Token + JWT**，部分功能需要确认：

1. **在线人数统计**：需要确认Sa-Token配置是否开启 `is-share=true`
2. **踢人效果**：JWT无状态，踢人后用户仍可访问直到token过期
3. **历史记录**：需要新建登录日志表
