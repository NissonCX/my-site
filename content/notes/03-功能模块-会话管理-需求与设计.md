## 一、模块背景

### 1.1 模块定位

**会话管理模块通过菜单权限控制访问**，谁有会话管理的菜单，谁就能使用这些功能：
- 分配菜单时控制哪些角色可见
- 不再局限于"管理员"，家长、老师等也可以有会话管理菜单

**功能归属**：
- 大部分功能是**管理端**（由菜单权限控制，非仅限管理员）
- 仅有"用户登录会话信息"同时支持**用户端**查看自己的设备列表

### 1.2 项目介绍

- **项目**：重庆九龙坡未成年人保护项目
- **技术栈**：Sa-Token + JWT
- **当前token配置**：24小时固定有效期

### 1.3 项目现有会话管理机制

**Sa-Token配置（application-dev.yml）：**

```yaml
sa-token:
  token-name: Authorization     # 请求头字段名
  token-prefix: Bearer          # JWT前缀
  jwt-secret-key: 'd0!doc15415B0*4G0`'
  is-print: false
  is-log: false
  isReadCookie: false
  isReadBody: false
  # token有效期 s
  timeout: 86400                # 24小时
```

**现有API能力：**

```java
// 获取当前登录用户信息
StpUtil.getLoginIdAsString()              // 获取当前登录ID
StpUtil.getTokenValue()                    // 当前token值
StpUtil.getSession()                       // 当前用户session
StpUtil.isLogin()                          // 检查是否登录
StpUtil.kickout(loginId)                   // 踢人下线
StpUtil.kickout(loginId, token)            // 顶号
StpUtil.getTokenInfo()                     // token详细信息
StpUtil.searchTokenValue(keyword, start, size)  // 搜索token
```

## 二、功能需求分析

### 功能点209：用户登录会话信息

#### 需求描述
用户登录会话信息 - 查看当前用户的登录会话详情

#### 需求归属
⭐ **双端功能**（菜单权限控制）
- 管理端（有菜单权限）：查看指定用户的登录会话
- 用户端：查看自己的登录设备列表

#### 需求分析

**业务场景**：
- 用户想查看自己当前登录的状态
- 显示当前登录的设备（Web/App）
- 有菜单权限的人帮助用户排查登录问题

**需要获取的信息**：

```java
// Sa-Token可获取的信息
- token值 (StpUtil.getTokenValue())
- 登录时间 (session.getCreateTime())
- 最后活跃时间 (需要自行记录)
- 用户ID (StpUtil.getLoginIdAsString())

// 需要扩展记录的信息
- IP地址 (通过ServletUtils获取)
- 设备类型 (PC/Mobile/App)
- 浏览器/操作系统信息
- 登录来源 (Web/小程序/App)
```

### 功能点210：设置会话有效期

#### 需求描述
设置会话有效期 - 修改token的有效时长

#### 需求归属
**管理端**（菜单权限控制）

#### 需求分析

**业务场景**：
- 有菜单权限的人统一修改登录有效期
- 针对不同角色设置不同的有效期（如学生7天、老师3天）

**现状分析**：
- 当前配置：固定24小时（86400秒）
- 存储方式：JWT无状态，服务端不存储

**实现方案对比**：

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| A. 固定全局有效期 | 所有用户统一配置 | 简单 | 无法个性化 |
| B. 角色配置 | 不同角色不同有效期 | 灵活 | 需要配置表 |
| C. 用户自定义 | 用户可选择有效期 | 最灵活 | 实现复杂 |

**推荐方案B：角色配置 + 默认全局配置**

```yaml
sa-token:
  default-timeout: 86400      # 全局默认24小时
  role-configs:              # 角色级别配置
    - roleCode: student
      timeout: 604800        # 学生7天
    - roleCode: teacher
      timeout: 259200        # 老师3天
    - roleCode: admin
      timeout: 86400         # 管理员24小时
```

**登录时动态设置有效期**：

```java
public void login(String username, String password) {
    // 1. 校验账号密码
    // 2. 获取用户角色
    List<String> roles = getUserRoles(username);

    // 3. 根据角色获取有效期
    long timeout = getTimeoutByRole(roles);

    // 4. 登录并设置有效期
    StpUtil.login(username);
    StpUtil.renewTimeout(timeout);  // 重置token有效期
}
```

### 功能点211：设置会话活跃状态

#### 需求描述
设置会话活跃状态 - 包括踢人下线、顶号、禁用会话等操作

#### 需求归属
**管理端**（菜单权限控制）

#### 需求分析

**业务场景**：
- 有菜单权限的人踢出违规用户
- 同一账号多设备登录顶号（挤掉其他设备）
- 禁用账户（禁止登录）

**Sa-Token现有能力**：

```java
// 踢人下线
StpUtil.kickout(loginId);           // 指定用户下线
StpUtil.kickout(loginId, token);    // 指定token下线

// 顶号（挤掉其他设备）
StpUtil.kickout(loginId, token);    // 最新登录挤掉之前的

// 检查是否在线
StpUtil.isLogin(loginId);           // 检查用户是否在线
StpUtil.getTokenListByLoginId(loginId);  // 获取用户所有token
```

### 功能点213：查看会话情况

#### 需求描述
查看会话情况 - 查看当前会话详情、历史会话记录

#### 需求归属
**管理端**（菜单权限控制）

#### 需求分析

**实现方案对比**：

| 方案 | 说明 | 适用场景 |
|------|------|----------|
| A. 实时查询Sa-Token | 查询内存/缓存中的token | 查看当前在线会话 |
| B. 记录登录日志 | 每次登录写入日志表 | 查看历史会话 |
| C. 两者结合 | 实时+历史都有 | 完整会话管理 |

**推荐方案C：实时+历史结合**

```sql
CREATE TABLE sys_login_log (
    id VARCHAR(64) PRIMARY KEY,
    user_id VARCHAR(64),
    username VARCHAR(64),
    ip_address VARCHAR(64),
    device_type VARCHAR(32),
    login_time TIMESTAMP,
    logout_time TIMESTAMP,
    status VARCHAR(1),  -- 0正常 1踢出
    create_time TIMESTAMP
);
```

### 功能点214：统计在线数量

#### 需求描述
统计在线数量 - 实时统计当前在线用户数

#### 需求归属
**管理端**（菜单权限控制）

#### 需求分析

**Sa-Token获取在线数量**：

```java
// 需要配置 is-share=true
StpUtil.getTokenCount();           // 当前在线token数量
StpUtil.getUserCount();            // 当前在线用户数量
StpUtil.getLoginDeviceCount();     // 各设备在线数量
```

**统计维度**：

| 维度 | 接口 | 说明 |
|------|------|------|
| 当前在线人数 | `StpUtil.getUserCount()` | 实时在线用户数 |
| 今日登录人数 | 需记录 | 今日登录的总人数 |
| 最高在线人数 | 需定时统计 | 历史最高在线数 |

**实现方案**：

```java
// 1. 当前在线（Sa-Token直接获取）
public long getOnlineCount() {
    return StpUtil.getUserCount();
}

// 2. 今日登录（登录时计数）
RedisUtil.incr("login:count:today");

// 3. 最高在线（定时任务）
@Scheduled(cron = "0 * * * * ?")  // 每分钟
public void recordPeakOnline() {
    long current = StpUtil.getUserCount();
    Long max = RedisUtil.get("online:peak:" + DateUtil.today());
    if (current > max) {
        RedisUtil.set("online:peak:" + DateUtil.today(), current);
    }
}
```

## 三、综合设计建议

### 3.1 模块划分

```
会话管理模块（菜单权限控制）
├── 会话查询
│   ├── 查看当前登录信息（有菜单权限）
│   ├── 查看会话情况（有菜单权限）
│   └── 查看在线列表（有菜单权限）
├── 会话控制
│   ├── 设置会话有效期（有菜单权限）
│   ├── 设置会话活跃状态（有菜单权限）
│   └── 踢人/顶号（有菜单权限）
├── 会话统计
│   └── 统计在线数量（有菜单权限）
└── 可视化展示（有菜单权限）

用户端
└── 我的设备列表（查看自己的登录设备）
```

### 3.2 接口路径规划

| 功能 | 路径 | 方法 | 归属 |
|------|------|------|------|
| 查看在线列表 | `/admin/session/online-list` | GET | 有菜单权限 |
| 查看会话统计 | `/admin/session/statistics` | GET | 有菜单权限 |
| 踢人下线 | `/admin/session/kickout/{userId}` | DELETE | 有菜单权限 |
| 顶号 | `/admin/session/replace/{userId}` | DELETE | 有菜单权限 |
| 设置会话有效期 | `/admin/session/timeout` | PUT | 有菜单权限 |
| 查看指定用户会话 | `/admin/session/user/{userId}` | GET | 有菜单权限 |
| 查看我的设备 | `/app/session/my-devices` | GET | 用户端 |
| 查看我的当前会话 | `/app/session/current` | GET | 用户端 |

### 3.3 设计原则

**核心思路：通过菜单权限控制访问**
- 不再区分"管理员"或"普通用户"
- 谁有会话管理菜单，谁就能使用这些功能
- 分配菜单时控制角色可见性

### 3.4 建议开发顺序

| 优先级 | 功能 | 原因 |
|--------|------|------|
| P0 | 查看在线列表 | 基础功能 |
| P0 | 踢人下线 | 核心功能 |
| P1 | 设置会话有效期 | 配置登录策略 |
| P1 | 查看会话统计 | 辅助功能 |
| P2 | 顶号 | 增强功能 |
| P2 | 历史登录记录 | 可选，后续迭代 |
| P3 | 可视化界面 | 前端配合 |

### 3.5 技术风险

1. **JWT无状态**：无法实时让token失效，踢人后用户仍可访问直到token过期
   - 解决：使用"黑名单"机制或改为有状态token

2. **在线人数统计**：Sa-Token的getUserCount()需要配置is-share=true
   - 需要确认当前配置是否已开启

3. **登录日志**：大量用户会产生大量日志
   - 解决：定期归档或使用ES存储
